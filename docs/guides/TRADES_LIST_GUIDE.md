# 交易明细功能说明

## 功能概述

在策略推演结果中新增交易明细展示，显示每一笔买入和卖出的详细信息。

## 功能特性

### 1. 交易记录

所有策略现在都会记录每笔交易的详细信息：

- **买入日期**：买入操作的日期
- **买入价格**：买入时的价格
- **买入金额**：买入的总金额
- **卖出日期**：卖出操作的日期
- **卖出价格**：卖出时的价格
- **卖出金额**：卖出的总金额
- **盈亏金额**：盈亏的绝对金额（正数为盈利，负数为亏损）
- **盈亏比例**：盈亏的百分比（正数为盈利，负数为亏损）
- **数量**：交易的份额数量

### 2. 交易明细表格

在策略推演页面，回测结果下方会显示交易明细表格，包含以下列：

| 列名 | 说明 | 格式 |
|------|------|------|
| # | 交易序号 | 1, 2, 3... |
| 买入日期 | 买入操作日期 | YYYY-MM-DD |
| 买入价格 | 买入时价格 | 保留3位小数 |
| 买入金额 | 买入总金额 | 保留2位小数 |
| 卖出日期 | 卖出操作日期 | YYYY-MM-DD |
| 卖出价格 | 卖出时价格 | 保留3位小数 |
| 卖出金额 | 卖出总金额 | 保留2位小数 |
| 盈亏金额 | 交易盈亏 | 保留2位小数，绿色盈利/红色亏损 |
| 盈亏比例 | 交易盈亏百分比 | 保留2位小数，绿色盈利/红色亏损 |
| 数量 | 交易份额 | 整数 |

### 3. 交易统计

在回测结果中显示：

- **交易总次数**：所有买卖对的总数
- **盈利次数**：盈利的交易次数
- **亏损次数**：亏损的交易次数
- **胜率**：盈利交易占比

## 使用方法

### 前端使用

1. 访问策略推演页面：`http://localhost:8080/backtest`

## 交易记录说明

### 买卖配对

每笔交易是一个完整的"买入-卖出"配对：
- 记录买入时的日期、价格、金额
- 记录卖出时的日期、价格、金额
- 计算盈亏金额和盈亏比例

### 盈亏计算

- **盈亏金额** = 卖出金额 - 买入金额
- **盈亏比例** = (卖出金额 / 买入金额 - 1) * 100%

### 颜色标识

- **绿色**：盈利（profit > 0）
- **红色**：亏损（profit < 0）

## 注意事项

1. **交易数量**：交易数量取决于策略的买卖信号频率
2. **手续费**：交易金额已包含手续费（默认0.03%）
3. **滑点**：回测未考虑滑点，实际交易可能会有差异
4. **未平仓**：如果有未平仓的持仓，不计入trades_list
5. **数据完整性**：确保ETF有足够的历史数据以获得完整的交易记录

## 实现细节

### 策略修改

所有策略类（MACD、布林带、组合策略）都添加了：

1. `trades_log` 列表：存储交易记录
2. `current_trade` 字典：记录当前持仓信息
3. `notify_order` 方法：监听订单完成事件，记录买卖信息
4. `get_trades_log` 方法：返回所有交易记录

### 回测引擎修改

`BacktestEngine.run_backtest` 方法：

1. 提取策略的 `get_trades_log()` 方法
2. 将交易记录添加到 `analyzer_results['trades_list']`
3. 返回包含交易明细的完整回测结果

### 前端修改

`Backtest.vue` 页面：

1. 添加交易明细表格卡片
2. 显示所有交易记录
3. 格式化显示价格、金额、盈亏等信息
4. 使用颜色标识盈利/亏损
