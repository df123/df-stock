# 图表功能测试指南

## 测试环境

- 前端开发服务器：`http://localhost:8080`
- 后端API服务器：`http://localhost:8000`
- 测试页面：策略筛选页面（Screening.vue）

## 测试前准备

1. 确保前端和后端服务都在运行
2. 打开浏览器访问：`http://localhost:8080`
3. 导航到"策略筛选"页面
4. 打开浏览器开发者工具（F12），查看Console和Network标签

---

## 测试项目

### 1. 基础功能测试

#### 测试步骤
1. 在策略筛选页面，点击"筛选"按钮
2. 等待筛选结果加载完成
3. 点击任意一个ETF行（例如：510300）
4. 等待图表对话框打开

#### 预期结果
- ✅ 图表对话框正常打开
- ✅ 显示图表控制面板（主图表、MACD、RSI、KDJ复选框）
- ✅ 默认显示主图表和MACD图表
- ✅ 图表正常渲染，显示价格线、布林带、MACD指标
- ✅ 图表图例正常显示
- ✅ 无控制台错误

#### 测试数据验证
- 主图表应显示：收盘价线、布林带上轨、布林带中轨、布林带下轨
- MACD图表应显示：MACD快线、慢线（信号线）、柱状图

---

### 2. 线条隐藏/显示功能测试

#### 测试步骤
1. 在主图表中，点击图例中的"布林带上轨"
2. 观察图表变化
3. 再次点击"布林带上轨"
4. 观察图表变化
5. 重复测试其他线条（布林带中轨、布林带下轨、收盘价）
6. 在MACD图表中，测试MACD快线、慢线、柱状图的隐藏/显示

#### 预期结果
- ✅ 点击图例后，对应的线条立即隐藏/显示
- ✅ 图例状态正确更新（选中/未选中）
- ✅ 线条可见性状态正确同步
- ✅ 隐藏线条后，其他线条保持正常显示
- ✅ 控制台输出图例选择改变的事件信息

#### 控制台验证
打开浏览器控制台，应该看到类似以下日志：
```
图例选择改变: {chartId: "screening-main", chartType: "main", indicatorId: "bollinger_upper", visible: false}
```

### 3. 数据点提示功能测试

#### 测试步骤
1. 将鼠标悬停在主图表上
2. 移动鼠标到不同的数据点
3. 观察提示框内容和位置
4. 在MACD图表中重复测试

#### 预期结果
- ✅ 鼠标悬停时显示数据点提示框
- ✅ 提示框包含：日期、收盘价、各指标值
- ✅ 提示框格式美观，包含颜色标识
- ✅ 提示框位置自动调整，避免超出边界
- ✅ 提示框显示所有可见线条的数据

#### 提示框内容示例

### 4. 多种指标显示测试

#### 测试步骤
1. 在图表控制面板中，取消勾选"MACD"
2. 勾选"RSI"
3. 观察图表变化
4. 勾选"KDJ"
5. 观察图表变化
6. 同时显示所有图表（主图表、MACD、RSI、KDJ）

#### 预期结果
- ✅ 取消勾选后，对应的图表立即隐藏
- ✅ 勾选后，对应的图表立即显示
- ✅ RSI图表正确显示RSI指标线
- ✅ KDJ图表正确显示K线、D线、J线
- ✅ RSI和KDJ图表的Y轴范围固定为0-100
- ✅ 所有图表同时显示时，布局正常
- ✅ 图表之间有适当的间距

#### RSI图表验证
- 应显示RSI指标线（蓝色）
- Y轴范围：0-100
- 可选：显示超买线（70）和超卖线（30）

#### KDJ图表验证
- 应显示K线（蓝色）、D线（黄色）、J线（红色）
- Y轴范围：0-100
- 可选：显示超买线（80）和超卖线（20）

## 附录：控制台错误参考

### 常见错误及解决方案

#### 错误1：图表DOM元素未找到
```
图表DOM元素未找到
```
**解决方案**：检查图表组件的ref是否正确绑定

#### 错误2：初始化图表失败
```
初始化图表失败: ...
```
**解决方案**：检查ECharts是否正确导入，检查图表配置

#### 错误3：更新图表失败
```
更新图表失败: ...
```
**解决方案**：检查数据格式是否正确，检查指标计算是否正常

#### 错误4：API请求失败
```
加载图表数据失败: ...
```
**解决方案**：检查后端API是否正常运行，检查网络连接
