# 增量更新ETF数据

## 功能说明

增量更新功能用于更新数据库中ETF的历史数据，仅获取每个ETF从最新日期到当前日期的新数据，而不是重新获取所有历史数据。这样可以大大减少数据获取时间和网络流量。

## 使用方法

### 1. 命令行使用

#### 更新所有ETF

```bash
cd /home/df/df-stock
python scripts/incremental_update.py
```

#### 更新指定ETF

```bash
python scripts/incremental_update.py --codes 510300 510500
```

#### 静默模式（不打印详细信息）

```bash
python scripts/incremental_update.py --quiet
```

### 2. Web界面使用

1. 访问 Web界面：http://localhost:8080
2. 进入"数据库管理"页面
3. 点击"增量更新"按钮
4. 等待更新完成（可能需要几分钟）
5. 更新完成后会自动刷新数据统计

### 3. API接口使用

#### 启动增量更新

```bash
curl -X POST http://localhost:8000/api/db/update/incremental
```

#### 更新指定ETF

```bash
curl -X POST "http://localhost:8000/api/db/update/incremental?codes=510300&codes=510500"
```

#### 查看更新状态

```bash
curl http://localhost:8000/api/db/update/status
```

## 工作原理

1. **获取ETF列表**：从数据库中获取所有ETF代码
2. **检查最新日期**：对于每个ETF，查询数据库中的最新日期
3. **获取新数据**：从最新日期的下一天开始获取新数据
4. **保存到数据库**：将新数据保存到数据库（使用INSERT OR REPLACE避免重复）

## 更新状态

增量更新任务的状态包括：

- `running`: 是否正在运行（true/false）
- `message`: 当前状态消息
- `progress`: 进度百分比（0-100）
- `current`: 当前进度（当前处理的ETF数量）
- `total`: 总数量（要更新的ETF总数）

## 进度显示

Web界面会实时显示更新进度，包括：

1. **进度条**: 显示当前更新进度（0-100%）
2. **状态消息**: 显示当前正在更新的ETF代码和进度
3. **详细信息**: 显示当前进度（如：进度: 123 / 1000）

更新完成后，进度条会变成绿色，并显示最终的更新结果。

## 策略筛选页面数字显示

策略筛选页面的所有数字字段都保留3位小数，包括：

- 最新价
- MACD快线
- MACD信号线
- 布林带上轨
- 布林带中轨
- 布林带下轨
- 布林带位置（百分比）

这样可以使数据显示更加精确和统一。

## 输出信息

更新完成后会输出：

- 总ETF代码数
- 成功更新的ETF数量
- 新增的记录数
- 失败的ETF数量
- 失败的ETF代码列表（如果有）

## 注意事项

1. **不要频繁更新**：建议每天更新一次，避免频繁调用API导致被限流
2. **后台运行**：增量更新是后台任务，可以继续使用其他功能
3. **网络依赖**：需要稳定的网络连接到新浪财经数据源
4. **数据库锁定**：更新期间可能会对数据库进行写操作，可能会影响查询性能
5. **错误处理**：如果某个ETF更新失败，不会影响其他ETF的更新

## 故障排查

### 问题1：更新任务一直显示"正在运行"

- 检查API服务日志：`tail -f logs/api.log`
- 可能是网络问题或API限流
- 等待几分钟后再次检查状态

### 问题2：更新失败数量很多

- 检查网络连接
- 可能是数据源暂时不可用
- 可以稍后再次尝试

### 问题3：新增记录数为0

- 可能是数据库中数据已是最新
- 或者是周末/节假日没有新数据
